ABSTRACT
一个好 skill 不是写出来的,是测出来的。Anthropic 的 skill-creator 把"凭感觉写提示词"推进成一条可运行的研发流水线:写草稿、跑测试、对比 baseline、让人审阅、量化评分,再把反馈喂回下一轮。
三个支点 / THREE PILLARS01—03
01

Baseline 对照

每个测试都跑"有 skill / 无 skill"两组,逼你回答:它到底比裸模型更好吗?

02

人类审阅

eval-viewer 把输出、评分、上轮反馈汇到一页,把人的判断做成流程节点。

03

触发优化

description 决定"有没有机会被用上";它把触发也做成可测试的任务。
