# 杂志编辑 / 瑞士国际 · 风格工作流

把「内容」长成这套理性编辑风的成品图/页。风格内核固定,只换内容。
有决策、可验证、可恢复的工作流,不是一次性 prompt。

## 第 0 步 · 读契约(每次必读)

风格全部固定值(色板/字体/网格系统/锚点/扁平铁律)在 → `references/style-core.md`。**先读它再动手**。

## 第 1 步 · 决策:做哪种产物

| 用户要的 | 产物 | 打开/读 | 形态 |
|---|---|---|---|
| 长图 / 题图 / 知识卡 / 海报 | **长图版** | `assets/changtu.html` | 竖向单图 |
| 多图 / 九宫格 / 轮播 | **多图版** | `assets/duotu.html` | 多张 3:4 |
| PPT / 幻灯片 / 演示 | **PPT 版** | `assets/deck.html` | 16:9 多页 |
| 配图提示词(套版图位,固定构成) | **插图提示词** | `references/illustration-prompts.md` | 中+英 |
| 文章插画 / 题图(不限版式,AI 自由构图) | **文章插画** | `references/article-illustration.md` | 中+英(只锁风格) |
| 视频 / 短视频 / 动态 | **视频版** | `references/video-prompts.md` | 中+英 |

每个 HTML 模板**顶部注释**写了该模式专属用法。打开即见,渐进式上下文。

## 第 2 步 · 生产

1. **理内容层级**:定出 眉标 / 大标题(哪个词点强调色)/ 2~4 个编号要点 / 一句金句。瑞士风靠层级,不靠装饰。
2. **完整复制对应模板**到目标位置(文件名含主题+日期)。
3. **只改内容**:眉标、标题、编号要点、正文、强调词、几何构成。网格/字体/留白/单一强调色照旧。
4. **守留白与对齐**:别填满,所有元素咬网格对齐。

> 可恢复:产物落盘 `.html`,中断重开即续;改某张直接编辑并重跑自检。

## 第 3 步 · 自检(必做)

```bash
python scripts/check_style.py 你的产出.html
```
验证:浅纸底 + 近黑文字 + **强调色至多一个**、三套字体(Archivo + Inter + Noto Sans SC)、网格布局、细分割线;并对**渐变/阴影/多强调色**等违反扁平克制的写法 FAIL。有 FAIL 改到通过再交付。
脚本外再人眼过软性目标(见 style-core):留白够、对齐齐、对比强、强调色省。

## 第 4 步 · 交接

- **导出成图**:长图/多图 → 截图(2x DPI);PPT → 打印存 PDF。自动化截图交接给浏览器/截图类工具。
- **真·可编辑 PPT**:坚持要 `.pptx` 时,把大纲交接给 `pptx` skill。
- **出图 / 出视频**:把 `illustration-prompts.md` / `article-illustration.md` / `video-prompts.md` 的提示词交给 AI 工具。

## 硬限制(仅此)

- 内容/数据来自用户输入,不杜撰。
- 扁平铁律:无渐变、无阴影、无 3D(自检会拦)。
- 强调色至多一个(瑞士风定义);可换色相,见 style-core。

其余皆为软性指导(见 `references/style-core.md`),理解原因后灵活权衡。
