# Anthropic 手绘极简 · 风格工作流

把「内容」长成这套极简手绘风的成品图/页。风格内核固定,只换内容。
这是一条**有决策、可验证、可恢复**的工作流,不是一次性 prompt。

## 第 0 步 · 读契约(每次必读)

风格全部固定值(三色板/字体/手绘滤镜/标注语言/扁平铁律/主体公式)在 → `references/style-core.md`。
**先读它再动手**。本文件只管流程与决策。

## 第 1 步 · 决策:做哪种产物

按用户意图选一种(或并行);每种只读它对应的一个文件:

| 用户要的 | 产物 | 打开/读 | 形态 |
|---|---|---|---|
| 长图 / 公众号配图 / 知识卡 / 示意图 / 海报 | **长图版** | `assets/changtu.html` | 竖向单图,导出 1 张长 PNG |
| 多图 / 九宫格 / 轮播 / 小红书 / 一组卡 | **多图版** | `assets/duotu.html` | 多张 3:4 卡,各自导出 |
| PPT / 幻灯片 / 演示 | **PPT 版** | `assets/deck.html` | 16:9 多页,导出 PDF |
| 配图提示词(套版图位用,固定主体公式) | **插图提示词** | `references/illustration-prompts.md` | 中+英 AI 提示词 |
| 文章插画 / 文章配图 / 题图(不限版式,AI 自由构图) | **文章插画** | `references/article-illustration.md` | 中+英 AI 提示词(只锁风格) |
| 视频 / 短视频 / 动态 / 白板讲解动画 | **视频版** | `references/video-prompts.md` | 中+英 AI 视频提示词(图生/文生) |

每个 HTML 模板**顶部注释**写了该模式的专属用法(改哪里、张数、导出方式)——打开即见,渐进式上下文,无需别处再记。

> 为什么是 HTML 不是 .pptx:手绘线的不完美感靠 SVG 位移滤镜 + 不均匀圆角实现,原生 PowerPoint 做不出。HTML 完整还原且能导出 PNG/PDF。用户坚持要可编辑 .pptx 时见「第 4 步 · 交接」。

## 第 2 步 · 生产

1. **想清核心意象**:这版要表达的"一个动作意象"(掌控趋势 / 调整模型 / 拣选节点 / 连接想法),决定主插画画什么(几何图表 + 手)。
2. **完整复制对应模板**到目标位置(默认用户工作目录,文件名含主题+日期)。
3. **只改内容**:标题(可给关键词加手绘圈/下划线)、正文、卡片文字、主插画的图表形态与标签。三色/字体/`#sketch` 滤镜/扁平铁律照旧。
4. **主插画**:模板内置"手调整上升折线"的 SVG 范例,改造它(换图表类型、换手的动作)即可;或把整块 `<svg class="hero">` 换成 `<img>`(用 `illustration-prompts.md` 出的图)。

> 可恢复:产物是落盘 `.html`,中断重开即续;改某张直接编辑该文件并重跑自检。

## 第 3 步 · 自检(出图质量闸门,必做)

```bash
python scripts/check_style.py 你的产出.html
```
它客观验证:**三色齐全、三套字体、`#sketch` 手绘滤镜、含手绘 SVG、米白卡片**;并对**违反扁平铁律的阴影/渐变/3D**直接 FAIL,对疑似杂色 WARN。**有 FAIL 改到通过再交付**。
脚本之外再人眼过软性目标(见 style-core「软性美学」):留白够、线够粗、有手作不完美感、一图一意。

## 第 4 步 · 交接

- **导出成图**:长图/多图 → 浏览器对 `.poster`/`.card` 截图(2x DPI);PPT → 打印存 PDF。需自动化截图交接给浏览器/截图类工具(webapp-testing / 截图 MCP)。
- **真·可编辑 PPT**:用户坚持要 `.pptx`,说明会丢失手绘质感后,把内容大纲交接给 `pptx` skill。
- **出图**:把插图提示词交给 AI 出图工具(即梦/MJ/SD),回填 `<img src>`。
- **出视频**:把视频提示词交给 AI 视频工具(即梦/可灵/海螺/Runway/Sora);**图生视频**优先用已出的长图/插画做静帧,白板手绘风最稳。

## 硬限制(仅此)

- 内容/事实/数据来自用户输入,不杜撰。
- 扁平铁律:不出现阴影、渐变、3D(违背风格定义,自检会拦)。
- 配色可换,但守规则:低饱和、≤3 色、近黑墨线 + 浅卡片(预设见 style-core)。不上鲜艳色;业务硬性指定的单一品牌强调色除外,且只点一处、仍偏低饱和。

其余皆为软性指导(见 `references/style-core.md`),理解原因后灵活权衡。
