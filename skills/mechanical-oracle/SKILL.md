# 电子梦呓 · 风格工作流

把「内容」长成统一风格的成品图/页。风格内核固定,只换内容——这样同一账号才成系列。
这是一条**有决策、可验证、可恢复**的工作流,不是一次性 prompt。

## 第 0 步 · 读契约(每次必读)

风格的全部固定值(色板/字体/质感代码/四锚点/可爱形象契约/软性美学)在 → `references/style-core.md`。
**先读它再动手**。本文件不重复这些内容,只管流程与决策。

## 第 1 步 · 决策:做哪种产物

按用户意图选一种(或并行多种);每种只读它对应的那一个文件,别全读:

| 用户要的 | 产物 | 打开/读 | 形态 |
|---|---|---|---|
| 长图 / 公众号配图 / 知识卡 / 海报 / 封面 | **长图版** | `assets/changtu.html` | 竖向单图,860 宽,导出 1 张长 PNG |
| 多图 / 九宫格 / 轮播 / 小红书 / 一组图 | **多图版** | `assets/duotu.html` | 多张 3:4 卡,各自导出 1 张 |
| PPT / 幻灯片 / 演示 | **PPT 版** | `assets/deck.html` | 16:9 多页,导出 PDF |
| 配图提示词(套版图位用,固定主体公式) | **插图提示词** | `references/illustration-prompts.md` | 中+英 AI 提示词 |
| 文章插画 / 文章配图 / 题图(不限版式,AI 自由构图) | **文章插画** | `references/article-illustration.md` | 中+英 AI 提示词(只锁风格) |
| 视频 / 短视频 / 动态 / 动态封面 / 片头 | **视频版** | `references/video-prompts.md` | 中+英 AI 视频提示词(图生/文生) |

不确定就问一句用户偏好;长图与多图常一起要(同一套内容两种分发)。
每个 HTML 模板**顶部注释**写了该模式的专属用法(张数、改哪里、导出方式)——打开即见,这就是渐进式上下文,无需别处再记。

> 为什么是 HTML 不是 .pptx:本风格靠 SVG 位移滤镜/半调网点/纸颗粒,原生 PowerPoint 做不出。HTML 完整还原且能导出 PNG/PDF 塞进任何渠道。用户坚持要可编辑 .pptx 时,见「第 4 步 · 交接」。

## 第 2 步 · 生产

1. **认领内容**:理清这一版要表达的主题与重点。
2. **完整复制对应模板**到目标位置(默认放用户的工作目录,文件名含主题+日期,便于复跑与回溯)。
3. **只改内容**:期号、标题(只让一个字变异)、正文、金句、插画位。色板/字体/滤镜/锚点结构照旧。
4. **插画**:模板已内置可爱主角 SVG,开箱即用;要换 AI 图就把 `<svg class="hero/masc">` 换成 `<img>`,并把 `references/illustration-prompts.md` 的中+英提示词交给用户去出图。

> 可恢复:产物是落盘的 `.html`,中断后重开即续;要改某张,直接编辑那个文件并重跑自检,无需从头。

## 第 3 步 · 自检(出图质量闸门,必做)

写完任何 HTML,跑:

```bash
python scripts/check_style.py 你的产出.html
```

它客观验证:六色板齐全、两套字体、warp 滤镜、纸颗粒、半调网点、刊头条、含变异字的标题；并对杂色、变异过多、瘆人词给 WARN。**有 FAIL 必须改到通过再交付**;WARN 逐条确认。
脚本之外再人眼过一遍软性目标(见 style-core「软性美学」):一处变异、正文规整、主角可爱不诡异、留白够。

## 第 4 步 · 交接

- **导出成图**:长图/多图 → 浏览器对 `.poster`/`.card` 截图(2x DPI);PPT → 打印存 PDF。需要自动化截图时,交接给浏览器/截图类工具(如 webapp-testing / 截图 MCP)按元素导出。
- **真·可编辑 PPT**:用户坚持要 `.pptx`,先说明会丢失怪诞质感,再把内容大纲交接给 `pptx` skill 生成(视觉退化版)。
- **出图**:把插图提示词交给用户的 AI 出图工具(即梦/MJ/SD),回填 `<img src>`。
- **出视频**:把视频提示词交给 AI 视频工具(即梦/可灵/海螺/Runway/Sora);**图生视频**优先用已出的长图/插画做静帧,风格最稳。

## 硬限制(仅此)

- 内容/事实/数据来自用户输入,不杜撰;数据缺口如实标注。
- 不输出诡异/恐怖形象(违反可爱契约,且不适合社交媒体)。

其余皆为软性指导(见 `references/style-core.md`),理解原因后可灵活权衡。
